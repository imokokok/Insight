# 预言机单独页面设计专业审查与改进建议 Spec

## Why

用户请求对十个预言机（Chainlink, Pyth, API3, UMA, Tellor, Chronicle, DIA, RedStone, BandProtocol, WINkLink）单独页面的设计进行专业审查，发现虽然页面采用了统一的设计模式，但仍存在多个可以改进的地方，包括用户体验、性能优化、代码质量、功能完整性等方面的问题。

## What Changes

- 增强用户体验：添加数据过期提示、离线状态处理、快捷键支持
- 改进性能：优化数据缓存策略、添加页面级性能监控
- 提升功能完整性：添加数据对比功能、历史数据导出、自定义时间范围
- 统一代码质量：修复不一致的代码模式、提取共享逻辑
- 增强可访问性：添加键盘导航、屏幕阅读器支持
- 改进错误处理：添加错误边界、更友好的错误提示

## Impact

- Affected specs: 所有预言机单独页面功能
- Affected code:
  - `src/app/[locale]/chainlink/` 目录下的所有文件
  - `src/app/[locale]/pyth/` 目录下的所有文件
  - `src/app/[locale]/api3/` 目录下的所有文件
  - `src/app/[locale]/uma/` 目录下的所有文件
  - `src/app/[locale]/tellor/` 目录下的所有文件
  - `src/app/[locale]/chronicle/` 目录下的所有文件
  - `src/app/[locale]/dia/` 目录下的所有文件
  - `src/app/[locale]/redstone/` 目录下的所有文件
  - `src/app/[locale]/band-protocol/` 目录下的所有文件

  - `src/components/oracle/` 目录（共享组件）
  - `src/hooks/` 目录（共享hooks）

## 发现的设计问题与改进建议

### 问题 1: 缺少数据新鲜度指示器（高优先级）

**位置**: 所有预言机的 Hero 组件
**当前状态**: 只显示 `lastUpdated` 时间，但没有直观的新鲜度指示
**问题描述**:

- 用户无法快速判断数据是否过期
- 没有视觉提示表明数据的新鲜程度
- 在数据过期时没有自动刷新或提示

**改进建议**:

- 添加数据新鲜度指示器（绿色=新鲜，黄色=即将过期，红色=已过期）
- 实现数据过期自动刷新机制
- 添加"数据已过期，点击刷新"的提示

**示例实现**:

```typescript
const getDataFreshness = (lastUpdated: Date | null): 'fresh' | 'stale' | 'expired' => {
  if (!lastUpdated) return 'expired';
  const ageInMinutes = (Date.now() - lastUpdated.getTime()) / 60000;
  if (ageInMinutes < 5) return 'fresh';
  if (ageInMinutes < 30) return 'stale';
  return 'expired';
};
```

### 问题 2: 缺少离线状态处理（高优先级）

**位置**: 所有预言机页面
**当前状态**: 没有处理网络离线状态
**问题描述**:

- 用户在离线时无法感知
- 没有离线状态提示
- 离线时刷新按钮仍然可用，会导致错误

**改进建议**:

- 添加网络状态检测 hook
- 在离线时显示离线提示横幅
- 禁用或隐藏刷新按钮
- 使用 Service Worker 缓存数据以支持离线访问

### 问题 3: 移动端菜单状态管理重复（中优先级）

**位置**: 所有预言机的 `page.tsx` 文件
**当前状态**: 每个页面都独立管理 `isMobileMenuOpen` 状态
**问题描述**:

```typescript
const [isMobileMenuOpen, setIsMobileMenuOpen] = useState(false);
```

- 代码重复，违反 DRY 原则
- 如果需要添加新功能（如菜单打开时的背景滚动锁定），需要修改所有文件

**改进建议**:

- 创建 `useMobileMenu` 共享 hook
- 包含菜单状态、打开/关闭逻辑、滚动锁定等功能

### 问题 4: RedStone 页面客户端实例创建位置不当（高优先级）

**位置**: `redstone/page.tsx` 第 26 行
**当前状态**:

```typescript
const redstoneClient = new RedStoneClient();
```

**问题描述**:

- 客户端实例在页面组件外部创建
- 可能导致服务端渲染问题
- 违反了其他预言机的模式（客户端实例应在 hook 中创建）

**改进建议**:

- 将客户端实例创建移到 `useRedStonePage` hook 中
- 使用单例模式确保只创建一个实例

### 问题 5: API3 页面数据映射在组件中进行（中优先级）

**位置**: `api3/page.tsx` 第 58-66 行
**当前状态**:

```typescript
const networkStats = {
  activeNodes: airnodeStats?.activeAirnodes ?? 0,
  dataFeeds: dapiCoverage?.totalDapis ?? 0,
  // ...
};
```

**问题描述**:

- 数据映射逻辑在页面组件中，而不是在 hook 中
- 违反了关注点分离原则
- 其他预言机都在 hook 中处理数据映射

**改进建议**:

- 将 `networkStats` 计算移到 `useAPI3Page` hook 中
- 保持与其他预言机的一致性

### 问题 6: 缺少错误边界（高优先级）

**位置**: 所有预言机页面
**当前状态**: 没有错误边界包裹页面内容
**问题描述**:

- 组件内部的 JavaScript 错误会导致整个页面崩溃
- 用户看到白屏或错误堆栈
- 没有优雅的错误恢复机制

**改进建议**:

- 添加 `OraclePageErrorBoundary` 组件
- 包裹每个预言机页面的内容
- 提供友好的错误提示和重试按钮

### 问题 7: 部分组件缺少 isLoading 传递（中优先级）

**位置**: 多个预言机的 `page.tsx` 文件
**当前状态**: 部分视图组件没有传递 `isLoading` 属性
**示例**:

```typescript
// Chainlink - 缺少 isLoading
case 'nodes':
  return <ChainlinkNodesView />;
case 'data-feeds':
  return <ChainlinkDataFeedsView />;
```

**问题描述**:

- 子组件无法正确显示加载状态
- 用户体验不一致

**改进建议**:

- 确保所有视图组件都接收并使用 `isLoading` 属性
- 或者使用骨架屏组件作为默认加载状态

### 问题 8: 缺少 SEO 元数据（中优先级）

**位置**: 所有预言机页面
**当前状态**: 没有设置页面标题、描述等 SEO 元数据
**问题描述**:

- 搜索引擎无法正确索引页面
- 社交媒体分享时缺少预览信息
- 浏览器标签显示默认标题

**改进建议**:

- 为每个预言机页面添加 `metadata` 导出
- 设置动态页面标题（如 "Chainlink 价格 | Insight"）
- 添加 Open Graph 和 Twitter Card 元数据

### 问题 9: 缺少键盘快捷键支持（低优先级）

**位置**: 所有预言机页面
**当前状态**: 没有键盘快捷键支持
**问题描述**:

- 高级用户无法使用键盘快速导航
- 无法使用快捷键刷新数据
- 可访问性不足

**改进建议**:

- 添加 `useKeyboardShortcuts` hook
- 支持常见快捷键：
  - `R` - 刷新数据
  - `1-7` - 切换 Tab
  - `E` - 导出数据
  - `?` - 显示快捷键帮助

### 问题 10: 缺少数据对比功能（中优先级）

**位置**: 所有预言机页面
**当前状态**: 没有与其他预言机对比的功能
**问题描述**:

- 用户无法快速比较不同预言机的价格差异
- 需要在多个页面之间切换
- 没有价格偏差分析

**改进建议**:

- 在 Hero 区域添加"与其他预言机对比"按钮
- 显示当前预言机价格与市场平均价格的偏差
- 提供跳转到跨预言机对比页面的链接

### 问题 11: 缺少历史数据导出功能（中优先级）

**位置**: 所有预言机页面
**当前状态**: 有 `exportData` 函数，但功能不明确
**问题描述**:

- 用户无法导出历史价格数据
- 无法选择导出时间范围
- 无法选择导出格式（CSV, JSON, Excel）

**改进建议**:

- 添加完整的导出功能面板
- 支持多种导出格式
- 支持自定义时间范围
- 添加批量导出功能

### 问题 12: 缺少自定义时间范围选择（中优先级）

**位置**: 所有预言机的 MarketView 组件
**当前状态**: 可能只有预设的时间范围选项
**问题描述**:

- 用户无法选择自定义的时间范围
- 无法查看特定历史事件期间的价格数据
- 灵活性不足

**改进建议**:

- 添加日期范围选择器
- 支持预设时间范围和自定义时间范围
- 保存用户偏好

### 问题 13: 缺少价格预警快速设置（中优先级）

**位置**: 所有预言机的 Hero 组件
**当前状态**: 没有快速设置价格预警的入口
**问题描述**:

- 用户需要导航到单独的预警页面
- 无法在查看价格时快速设置预警
- 用户体验不连贯

**改进建议**:

- 在 Hero 区域添加"设置预警"按钮
- 点击后弹出快速预警设置对话框
- 预填充当前预言机和价格信息

### 问题 14: 缺少数据质量指标（中优先级）

**位置**: 所有预言机页面
**当前状态**: 没有显示数据质量指标
**问题描述**:

- 用户无法判断数据源的可靠性
- 没有显示数据延迟、更新频率等信息
- 缺少数据完整性指标

**改进建议**:

- 添加数据质量面板
- 显示以下指标：
  - 数据延迟（毫秒）
  - 更新频率
  - 数据源数量
  - 历史数据完整性

### 问题 15: Chronicle 页面缺少 useTranslations 导入（低优先级）

**位置**: `chronicle/page.tsx`
**当前状态**: 文件中使用了 `t()` 函数，但没有导入 `useTranslations`
**问题描述**:

```typescript
// 缺少这行导入
import { useTranslations } from '@/i18n';
```

- 代码不一致
- 可能导致运行时错误

**改进建议**:

- 添加缺失的导入语句
- 确保所有页面都有相同的导入模式

### 问题 16: 缺少页面加载性能监控（低优先级）

**位置**: 所有预言机页面
**当前状态**: 没有性能监控
**问题描述**:

- 无法了解页面加载性能
- 无法识别性能瓶颈
- 无法优化用户体验

**改进建议**:

- 添加性能监控 hook
- 记录关键性能指标：
  - 首次内容绘制（FCP）
  - 最大内容绘制（LCP）
  - 首次输入延迟（FID）
  - 累积布局偏移（CLS）

### 问题 17: 缺少数据缓存策略说明（低优先级）

**位置**: 所有预言机的 hooks 文件
**当前状态**: 使用 React Query 的默认缓存策略
**问题描述**:

- 用户不知道数据何时会过期
- 没有明确的缓存时间配置
- 可能导致显示过期数据

**改进建议**:

- 为不同类型的数据设置不同的缓存时间
- 在 UI 中显示缓存状态
- 提供"强制刷新"选项

### 问题 18: 缺少可访问性优化（中优先级）

**位置**: 所有预言机页面
**当前状态**: 基本的可访问性支持
**问题描述**:

- 缺少 ARIA 标签
- 键盘导航不完整
- 屏幕阅读器支持不足

**改进建议**:

- 添加完整的 ARIA 标签
- 确保所有交互元素可通过键盘访问
- 添加屏幕阅读器友好的提示
- 支持高对比度模式

### 问题 19: 缺少页面级别的加载进度指示（低优先级）

**位置**: 所有预言机页面
**当前状态**: 只有全屏加载状态或无加载状态
**问题描述**:

- 用户无法了解数据加载进度
- 长时间加载时用户可能认为页面卡死

**改进建议**:

- 添加顶部进度条
- 显示加载步骤（如"正在获取价格数据..."）
- 支持部分加载显示

### 问题 20: 缺少用户偏好持久化（低优先级）

**位置**: 所有预言机页面
**当前状态**: 用户选择的 Tab、时间范围等没有持久化
**问题描述**:

- 刷新页面后用户设置丢失
- 每次都需要重新选择偏好
- 用户体验不佳

**改进建议**:

- 使用 localStorage 或 cookies 保存用户偏好
- 保存以下设置：
  - 上次选择的 Tab
  - 时间范围偏好
  - 图表类型偏好
  - 主题偏好

## ADDED Requirements

### Requirement: 数据新鲜度可视化

系统 SHALL 提供直观的数据新鲜度指示器，帮助用户快速判断数据是否过期。

#### Scenario: 数据新鲜度显示

- **WHEN** 用户查看预言机页面
- **THEN** 系统应显示数据新鲜度指示器（绿色/黄色/红色）
- **AND** 当数据过期时，应显示"数据已过期，点击刷新"提示
- **AND** 用户可以点击刷新按钮获取最新数据

### Requirement: 离线状态处理

系统 SHALL 正确处理网络离线状态，提供友好的用户提示。

#### Scenario: 用户离线时访问页面

- **WHEN** 用户网络连接断开
- **THEN** 系统应显示离线提示横幅
- **AND** 禁用刷新按钮或显示"离线状态"提示
- **AND** 显示缓存的数据（如果有）

### Requirement: 错误边界保护

系统 SHALL 使用错误边界包裹页面内容，防止组件错误导致页面崩溃。

#### Scenario: 组件内部发生错误

- **WHEN** 页面组件内部发生 JavaScript 错误
- **THEN** 错误边界应捕获错误
- **AND** 显示友好的错误提示页面
- **AND** 提供重试按钮

### Requirement: SEO 元数据配置

系统 SHALL 为每个预言机页面配置完整的 SEO 元数据。

#### Scenario: 搜索引擎索引页面

- **WHEN** 搜索引擎爬虫访问预言机页面
- **THEN** 系统应返回完整的元数据
- **AND** 包含页面标题、描述、关键词
- **AND** 包含 Open Graph 和 Twitter Card 元数据

### Requirement: 键盘快捷键支持

系统 SHALL 支持键盘快捷键，提高高级用户的使用效率。

#### Scenario: 用户使用键盘快捷键

- **WHEN** 用户按下快捷键（如 R 刷新）
- **THEN** 系统应执行相应操作
- **AND** 显示快捷键提示（如按下 ? 键）
- **AND** 快捷键不与浏览器快捷键冲突

### Requirement: 数据对比功能

系统 SHALL 提供预言机数据对比功能，帮助用户了解价格差异。

#### Scenario: 用户查看价格对比

- **WHEN** 用户点击"与其他预言机对比"按钮
- **THEN** 系统应显示与其他预言机的价格对比
- **AND** 显示价格偏差百分比
- **AND** 提供跳转到跨预言机对比页面的链接

### Requirement: 历史数据导出

系统 SHALL 提供完整的历史数据导出功能。

#### Scenario: 用户导出历史数据

- **WHEN** 用户点击导出按钮
- **THEN** 系统应显示导出配置面板
- **AND** 用户可以选择时间范围
- **AND** 用户可以选择导出格式（CSV, JSON, Excel）
- **AND** 导出文件应包含完整的历史价格数据

### Requirement: 用户偏好持久化

系统 SHALL 持久化用户的页面偏好设置。

#### Scenario: 用户刷新页面

- **WHEN** 用户刷新预言机页面
- **THEN** 系统应恢复用户上次的选择
- **AND** 包括 Tab 选择、时间范围、图表类型等
- **AND** 用户可以随时更改这些设置
