# RedStone 预言机页面专业评审规范

## Why

对当前 RedStone 预言机页面进行全面专业评估，识别功能覆盖差距，确保页面能够充分展示 RedStone 作为模块化预言机的核心能力和独特价值主张（Pull Model、混合模式、ERC-7412 标准、EigenLayer AVS 等）。

## What Changes

- 评估现有功能覆盖度与 RedStone 核心特性的匹配程度
- 识别缺失的核心功能模块
- 提供专业改进建议和优先级排序
- 提出数据真实性和实时性改进方案

## Impact

- Affected specs: RedStone 页面功能范围
- Affected code: `/src/app/[locale]/redstone/` 目录下的所有组件

## 数据真实性问题

### 当前问题

1. **Mock 数据依赖**: 大量使用硬编码的模拟数据
   - `SUPPORTED_CHAINS` - 链列表数据
   - `ECOSYSTEM_INTEGRATIONS` - 生态集成数据
   - `historicalRiskEvents` - 历史风险事件
   - `tvlTrendData` - TVL 趋势数据

2. **缺少实时数据源**:
   - 无 WebSocket 实时更新
   - 无链上数据获取
   - 无第三方 API 集成

3. **数据提供者信息不真实**:
   - 提供者列表为模拟数据
   - 声誉评分为随机生成
   - 数据点数为硬编码

### 建议改进

1. **集成 RedStone 官方 API**
   ```typescript
   const REDSTONE_API_ENDPOINTS = {
     prices: 'https://api.redstone.finance/prices',
     providers: 'https://api.redstone.finance/providers',
     stats: 'https://api.redstone.finance/stats',
     avatars: 'https://api.redstone.finance/avatars',
   };
   ```

2. **集成链上数据**
   - 使用 Etherscan API 获取交易数据
   - 使用 Dune Analytics 获取聚合数据
   - 使用 The Graph 进行链上查询

3. **实时更新机制**
   - WebSocket 连接到 RedStone 节点
   - 定期轮询更新关键指标
   - 缓存策略优化
