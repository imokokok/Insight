# 跨预言机比较页面数据深度专业分析

## Why

作为专业数据分析平台，跨预言机比较页面的数据深度直接影响用户对预言机价格数据质量的判断能力。本分析旨在系统评估当前页面在数据维度、统计深度、可视化呈现等方面的专业程度，识别数据深度的优势与不足，为平台数据能力建设提供参考。

## What Changes

- 对现有跨预言机比较页面的数据深度进行全面专业评估
- 分析数据维度覆盖情况（价格、时间、偏差、置信度等）
- 评估统计分析方法的专业性（标准差、方差、Z-Score、相关系数等）
- 检查可视化组件的数据表达深度
- 识别数据深度的优势与改进空间

## Impact

- Affected specs: 无现有 spec 受影响
- Affected code: 跨预言机比较页面相关组件

## ADDED Requirements

### Requirement: 数据维度覆盖分析
系统 SHALL 评估跨预言机比较页面的数据维度覆盖情况。

#### Scenario: 基础价格数据维度
- **WHEN** 分析价格数据维度
- **THEN** 应确认支持多预言机实时价格获取
- **AND** 应支持历史价格数据查询（1H/24H/7D/30D/90D/1Y/ALL）
- **AND** 应提供加权平均价格计算

#### Scenario: 时间维度分析
- **WHEN** 分析时间维度
- **THEN** 应支持多时间粒度（秒级新鲜度）
- **AND** 应提供数据更新时间戳
- **AND** 应支持历史时间范围筛选

#### Scenario: 偏差分析维度
- **WHEN** 分析偏差维度
- **THEN** 应计算各预言机价格与平均价格的偏差百分比
- **AND** 应支持偏差分级（优秀<0.1%/良好0.1-0.5%/较差>0.5%）
- **AND** 应提供偏差热力图可视化

#### Scenario: 统计指标维度
- **WHEN** 分析统计指标
- **THEN** 应提供平均值、最大值、最小值
- **AND** 应计算标准差和方差
- **AND** 应支持Z-Score异常值检测
- **AND** 应提供价格范围分析

### Requirement: 统计分析深度评估
系统 SHALL 评估页面使用的统计分析方法的专业性。

#### Scenario: 描述性统计
- **WHEN** 评估描述性统计
- **THEN** 应使用标准差衡量价格波动性
- **AND** 应使用方差分析价格离散程度
- **AND** 应计算加权平均值（基于置信度）

#### Scenario: 异常检测
- **WHEN** 评估异常检测能力
- **THEN** 应使用Z-Score识别异常值（|Z|>2）
- **AND** 应提供异常值可视化标记
- **AND** 应支持异常值快速定位

#### Scenario: 相关性分析
- **WHEN** 评估相关性分析
- **THEN** 应使用Pearson相关系数计算预言机间价格相关性
- **AND** 应提供相关性矩阵可视化
- **AND** 应支持相关性分级（极强/强/中等/弱/极弱）

#### Scenario: 分布分析
- **WHEN** 评估分布分析
- **THEN** 应提供箱线图展示价格分布（Min/Q1/Median/Q3/Max）
- **AND** 应识别离群值（IQR方法）
- **AND** 应支持四分位距（IQR）计算

### Requirement: 可视化数据深度评估
系统 SHALL 评估可视化组件的数据表达深度。

#### Scenario: 价格趋势图
- **WHEN** 评估价格趋势图
- **THEN** 应支持多预言机价格对比
- **AND** 应显示平均价格线
- **AND** 应提供±1σ和±2σ标准差区间
- **AND** 应支持图表缩放（0.5x-5x）

#### Scenario: 偏差热力图
- **WHEN** 评估偏差热力图
- **THEN** 应展示时间×预言机的偏差矩阵
- **AND** 应使用9级颜色梯度表示偏差程度
- **AND** 应提供偏差统计摘要（平均/最大/最小/异常数）
- **AND** 应支持预言机偏差排名

#### Scenario: 性能排名可视化
- **WHEN** 评估性能排名
- **THEN** 应使用多维度加权评分（响应时间30%/准确率30%/稳定性20%/数据源10%/支持链10%）
- **AND** 应提供各维度得分明细
- **AND** 应支持排名变化追踪

#### Scenario: 数据质量评分
- **WHEN** 评估数据质量评分
- **THEN** 应提供完整性评分（成功率/总数）
- **AND** 应提供可靠性评分（历史准确率/响应成功率）
- **AND** 应提供新鲜度评分（最后更新时间）

### Requirement: 数据深度优势识别
系统 SHALL 识别当前数据深度的优势。

#### Scenario: 统计方法优势
- **WHEN** 评估统计方法
- **THEN** 应确认使用了Z-Score异常检测
- **AND** 应确认使用了Pearson相关性分析
- **AND** 应确认使用了箱线图分布分析
- **AND** 应确认使用了加权平均计算

#### Scenario: 多维度对比优势
- **WHEN** 评估多维度对比
- **THEN** 应支持5个预言机同时对比
- **AND** 应提供4个主要分析标签页（概览/图表/快照/性能）
- **AND** 应支持多时间范围筛选

#### Scenario: 实时性优势
- **WHEN** 评估实时性
- **THEN** 应支持秒级数据新鲜度显示
- **AND** 应支持自动刷新（30秒/1分钟/5分钟）
- **AND** 应提供数据更新时间戳

### Requirement: 数据深度不足识别
系统 SHALL 识别当前数据深度的不足之处。

#### Scenario: 缺失的高级统计
- **WHEN** 评估高级统计缺失
- **THEN** 应识别缺乏滑动窗口统计
- **AND** 应识别缺乏趋势预测分析
- **AND** 应识别缺乏波动性指标（如ATR、Bollinger Bands）

#### Scenario: 缺失的链上数据
- **WHEN** 评估链上数据缺失
- **THEN** 应识别缺乏Gas费用对比
- **AND** 应识别缺乏链上更新延迟分析
- **AND** 应识别缺乏数据源透明度信息

#### Scenario: 缺失的历史分析
- **WHEN** 评估历史分析缺失
- **THEN** 应识别缺乏长期趋势分析（>1年）
- **AND** 应识别缺乏季节性模式识别
- **AND** 应识别缺乏历史异常事件标记

### ✅ 数据深度优势

#### 1. 统计方法专业度高 ⭐⭐⭐⭐⭐

**优势体现：**
- **Z-Score异常检测**：使用统计学标准方法（|Z|>2）识别异常值，并支持一键定位
- **Pearson相关性分析**：完整实现皮尔逊相关系数计算，支持相关性矩阵可视化
- **箱线图分布分析**：完整五数概括（Min/Q1/Median/Q3/Max）+ IQR离群值检测
- **加权平均计算**：基于置信度的加权平均，更符合实际应用场景

**专业评价：**
统计方法选择恰当，实现完整，达到了专业数据分析平台的标准。

#### 2. 多维度对比能力强 ⭐⭐⭐⭐⭐

**优势体现：**
- **多预言机支持**：同时对比5个主流预言机（Chainlink/Pyth/Band/UMA/API3）
- **多时间范围**：支持7个时间粒度（1H/24H/7D/30D/90D/1Y/ALL）
- **多标签页设计**：概览/图表/快照/性能四个维度全面分析
- **实时筛选**：支持偏差过滤、预言机过滤、时间范围过滤

**专业评价：**
对比维度全面，满足不同场景的分析需求。

#### 3. 实时性表现优秀 ⭐⭐⭐⭐⭐

**优势体现：**
- **秒级新鲜度**：精确到秒的数据年龄显示
- **自动刷新**：支持30秒/1分钟/5分钟自动刷新
- **实时标记**：最高/最低/异常值实时标记
- **快照对比**：支持历史快照保存和对比

**专业评价：**
实时性设计完善，满足实时监控需求。

#### 4. 可视化表达清晰 ⭐⭐⭐⭐⭐

**优势体现：**
- **标准差区间**：价格趋势图显示±1σ和±2σ区间，直观展示价格波动范围
- **热力图分级**：9级颜色梯度精确表达偏差程度
- **交互式提示**：丰富的Tooltip展示详细数据
- **响应式设计**：支持移动端和桌面端

**专业评价：**
可视化设计专业，数据表达清晰直观。

### ⚠️ 数据深度不足

#### 1. 缺乏高级时间序列分析 ⭐⭐

**不足之处：**
- **无滑动窗口统计**：缺乏动态滑动窗口的均值/标准差计算
- **无趋势预测**：缺乏基于历史数据的趋势预测能力
- **无季节性分析**：缺乏长期数据的季节性模式识别

**影响：**
用户无法进行更深入的时间序列分析，难以识别长期趋势。

**建议：**
- 添加滑动窗口统计（如7日移动平均）
- 集成简单预测模型（如线性回归）
- 支持长期趋势对比（月度/季度/年度）

#### 2. 缺乏链上数据深度 ⭐⭐

**不足之处：**
- **无Gas费用对比**：未展示各预言机的链上更新成本
- **无更新延迟分析**：缺乏链上更新到显示的延迟分析
- **无数据源透明度**：缺乏数据源的具体信息和可信度评估

**影响：**
用户无法全面评估预言机的链上性能和成本。

**建议：**
- 添加Gas费用对比功能
- 实现链上延迟监控
- 增强数据源透明度展示

#### 3. 缺乏高级波动性指标 ⭐⭐⭐

**不足之处：**
- **仅基础标准差**：缺乏ATR（平均真实波幅）、布林带等专业指标
- **无波动率指数**：缺乏类似VIX的波动率指数
- **无风险度量**：缺乏VaR（风险价值）等风险指标

**影响：**
专业用户无法进行深入的波动性分析。

**建议：**
- 添加ATR指标
- 实现布林带可视化
- 添加基础风险度量

#### 4. 数据质量评分深度不足 ⭐⭐⭐

**不足之处：**
- **指标简单**：仅完整性/准确性/新鲜度三个维度
- **缺乏权重配置**：用户无法自定义评分权重
- **缺乏历史趋势**：无数据质量历史趋势分析

**影响：**
数据质量评估不够全面，难以满足专业需求。

**建议：**
- 扩展评分维度（添加延迟、稳定性等）
- 支持权重自定义
- 添加质量趋势图
