# 市场概览功能专业审查 Spec

## Why

市场概览是项目的核心页面之一，需要从用户体验、功能完整性、性能、代码质量和可访问性等多个维度进行专业审查，提供改进建议。

## What Changes

- 识别用户体验问题并提供改进方案
- 识别功能缺失并建议补充
- 识别性能瓶颈并提供优化方案
- 识别代码质量问题并提供重构建议
- 识别可访问性问题并提供修复方案

## Impact

- Affected specs: 市场概览页面所有组件
- Affected code:
  - `src/app/[locale]/market-overview/` 目录下所有文件
  - `src/lib/services/marketData/` 数据服务
  - `src/components/ui/` 相关 UI 组件

## 发现的问题详情

### 一、用户体验问题 (高优先级)

#### 问题 1.1: 缺少数据加载骨架屏

**文件**: `ChartContainer.tsx`, `MarketStats.tsx`, `AssetsTable.tsx`

**现状**: 数据加载时只显示简单的 loading spinner，用户无法预知内容结构。

**建议**:

- 为 MarketStats 添加骨架屏，显示 6 个占位卡片
- 为 ChartContainer 添加图表骨架屏，显示占位网格
- 为 AssetsTable 添加表格骨架屏，显示 5-10 行占位数据

**影响**: 提升用户感知性能，减少等待焦虑

#### 问题 1.2: 图表类型切换缺少过渡动画

**文件**: `ChartContainer.tsx` 第 333-376 行

**现状**: 切换图表类型时直接替换，没有过渡效果。

**建议**:

- 添加淡入淡出动画 (fade transition)
- 使用 CSS transform 实现平滑过渡
- 考虑使用 framer-motion 或 CSS transitions

#### 问题 1.3: 缺少数据为空的状态处理

**文件**: `ChartRenderer.tsx`, `AssetsTable.tsx`

**现状**: 当 `sortedOracleData` 或 `assets` 为空数组时，图表显示空白或异常。

**建议**:

- 添加 EmptyState 组件，显示友好的空状态提示
- 提供刷新按钮引导用户重新加载数据
- 对于筛选结果为空，显示"无匹配数据"提示

#### 问题 1.4: 移动端图表体验不佳

**文件**: `ChartRenderer.tsx`

**现状**: 趋势图在移动端显示 10 条预言机数据线，过于拥挤。

**建议**:

- 移动端默认只显示 Top 5 预言机
- 提供"显示更多"按钮展开完整列表
- 考虑移动端使用简化的图表类型

#### 问题 1.5: 数据刷新缺少视觉反馈

**文件**: `MarketStats.tsx`

**现状**: 数据刷新后，数值直接变化，用户无法感知数据更新。

**建议**:

- 添加数值变化的高亮动画 (flash effect)
- 显示变化方向指示器 (上升/下降箭头)
- 考虑使用数字滚动动画效果

### 二、功能缺失问题 (中优先级)

#### 问题 2.1: 缺少数据筛选功能

**文件**: `MarketSidebar.tsx`, `AssetsTable.tsx`

**现状**: 用户无法按特定条件筛选预言机或资产数据。

**建议**:

- 在 MarketSidebar 添加筛选器：
  - 按市场份额筛选 (如 >5%)
  - 按 24h 变化筛选 (如正增长/负增长)
  - 按链数量筛选
- 在 AssetsTable 添加筛选器：
  - 按价格范围筛选
  - 按预言机筛选
  - 按变化幅度筛选

#### 问题 2.2: 缺少预言机详细对比功能

**文件**: `ChartContainer.tsx`

**现状**: 用户只能通过 linkedOracle 功能查看两个预言机的关联，无法进行详细对比。

**建议**:

- 添加"对比模式"功能，允许用户选择 2-3 个预言机
- 显示并排对比视图，包含关键指标对比
- 添加差异高亮显示

#### 问题 2.3: 缺少历史数据快照功能

**文件**: `useDataFetching.ts`

**现状**: 用户无法查看历史时间点的数据快照。

**建议**:

- 添加"历史快照"功能，允许用户选择特定日期
- 存储历史数据快照（可考虑本地存储或 API 支持）
- 显示历史数据与当前数据的对比

#### 问题 2.4: 导出功能格式有限

**文件**: `UnifiedExportSection.tsx`

**现状**: 导出只支持 CSV 和 JSON 格式。

**建议**:

- 添加 Excel (XLSX) 格式支持
- 添加 PDF 报告导出（包含图表截图）
- 添加图片导出（PNG/SVG）

#### 问题 2.5: 缺少预警设置入口

**文件**: `MarketHeader.tsx`

**现状**: 系统有预警功能，但在市场概览页面没有明显的入口。

**建议**:

- 在 MarketHeader 添加"设置预警"按钮
- 允许用户快速设置价格预警
- 显示当前活跃预警数量

### 三、性能问题 (中优先级)

#### 问题 3.1: 趋势图数据量大时性能下降

**文件**: `ChartRenderer.tsx` 第 373-574 行

**现状**: 趋势图在 1Y 时间范围下可能渲染数千个数据点，导致性能下降。

**建议**:

- 实现数据采样算法，根据时间范围动态调整数据点数量
- 1H: 全部数据点
- 24H: 每 5 分钟一个点
- 7D: 每 30 分钟一个点
- 30D+: 每小时一个点
- 使用 Web Worker 进行数据处理

#### 问题 3.2: AssetsTable 缺少虚拟滚动

**文件**: `AssetsTable.tsx`

**现状**: 当资产数量较多时，表格渲染可能变慢。

**建议**:

- 使用 react-window 或 react-virtualized 实现虚拟滚动
- 设置合理的缓冲区大小
- 保持滚动流畅性

#### 问题 3.3: 缺少数据缓存策略

**文件**: `useDataFetching.ts`

**现状**: 每次切换时间范围都会重新获取数据，即使数据可能相同。

**建议**:

- 实现基于时间范围的内存缓存
- 设置合理的缓存过期时间
- 使用 SWR 或 React Query 进行数据管理

#### 问题 3.4: 图表重渲染过于频繁

**文件**: `ChartContainer.tsx`, `ChartRenderer.tsx`

**现状**: 任何状态变化都可能导致整个图表重新渲染。

**建议**:

- 使用 React.memo 包装图表组件
- 拆分图表配置和数据处理逻辑
- 使用 useMemo 缓存计算结果

### 四、代码质量问题 (低优先级)

#### 问题 4.1: ChartContainer 组件职责过重

**文件**: `ChartContainer.tsx` (810 行)

**现状**: 单个组件包含图表类型切换、时间范围选择、工具栏、渲染逻辑等多种职责。

**建议**:

- 拆分为以下组件：
  - `ChartTypeSelector`: 图表类型选择器
  - `ChartToolbar`: 图表工具栏
  - `ChartTimeRangeSelector`: 时间范围选择器
  - `ChartContent`: 图表内容区域
- 使用组合模式组织组件

#### 问题 4.2: ChartRenderer 组件过于庞大

**文件**: `ChartRenderer.tsx` (877 行)

**现状**: 单个组件包含 9 种图表的渲染逻辑。

**建议**:

- 拆分为独立的图表组件：
  - `PieChartRenderer`
  - `TrendChartRenderer`
  - `BarChartRenderer`
  - 等...
- 使用策略模式或工厂模式管理图表渲染

#### 问题 4.3: 缺少错误边界

**文件**: `page.tsx`, `ChartContainer.tsx`

**现状**: 图表组件没有错误边界保护，一个图表出错可能导致整个页面崩溃。

**建议**:

- 为每个图表添加 ErrorBoundary
- 提供友好的错误提示和重试按钮
- 记录错误日志用于调试

#### 问题 4.4: 类型定义分散

**文件**: `types.ts`, `comparisonUtils.ts`, 各组件内部

**现状**: 类型定义分散在多个文件中，部分类型在组件内部定义。

**建议**:

- 集中管理类型定义在 `types.ts`
- 使用 TypeScript 的 namespace 或模块组织类型
- 添加 JSDoc 注释说明类型用途

#### 问题 4.5: 缺少单元测试

**文件**: 整个 `market-overview` 目录

**现状**: 关键计算逻辑和组件缺少单元测试覆盖。

**建议**:

- 为 `useMarketOverviewData` 添加测试
- 为 `marketStats` 计算逻辑添加测试
- 为图表组件添加快照测试
- 目标覆盖率: 80%+

### 五、可访问性问题 (中优先级)

#### 问题 5.1: 缺少键盘导航支持

**文件**: `ChartRenderer.tsx`, `MarketSidebar.tsx`

**现状**: 图表交互主要依赖鼠标，键盘用户无法正常使用。

**建议**:

- 添加 Tab 键导航支持
- 添加 Enter/Space 键激活支持
- 添加方向键在图表元素间导航
- 使用 `aria-activedescendant` 跟踪焦点

#### 问题 5.2: 缺少屏幕阅读器支持

**文件**: `ChartRenderer.tsx`

**现状**: 图表数据对屏幕阅读器不友好，视障用户无法获取信息。

**建议**:

- 为图表添加 `aria-label` 和 `aria-describedby`
- 提供数据表格作为替代视图
- 使用 `role="img"` 和 `aria-label` 描述图表内容
- 添加隐藏的数据摘要供屏幕阅读器读取

#### 问题 5.3: 颜色对比度问题

**文件**: `colors.ts`, `ChartRenderer.tsx`

**现状**: 部分颜色组合对比度可能不足，影响色盲用户。

**建议**:

- 使用 WCAG 2.1 AA 标准检查颜色对比度 (至少 4.5:1)
- 为色盲用户提供高对比度模式
- 使用图案或形状区分数据，不仅依赖颜色
- 考虑添加颜色盲模拟测试

#### 问题 5.4: 缺少焦点指示器

**文件**: `MarketSidebar.tsx`, `ChartContainer.tsx`

**现状**: 交互元素获得焦点时没有明显的视觉指示。

**建议**:

- 为所有可交互元素添加 `focus:ring` 样式
- 使用 Tailwind 的 `focus-visible` 伪类
- 确保焦点指示器颜色对比度足够

### 六、数据展示问题 (低优先级)

#### 问题 6.1: 缺少数据来源说明

**文件**: `MarketHeader.tsx`, `MarketStats.tsx`

**现状**: 用户不知道数据的来源和更新频率。

**建议**:

- 在 MarketHeader 添加数据来源说明
- 显示数据提供商信息
- 说明数据更新频率

#### 问题 6.2: 缺少数据准确性提示

**文件**: `MarketStats.tsx`

**现状**: 没有显示数据的置信度或误差范围。

**建议**:

- 为关键指标添加置信区间显示
- 显示数据延迟信息
- 添加数据质量评分

#### 问题 6.3: 数据单位不够明确

**文件**: `MarketStats.tsx`, `AssetsTable.tsx`

**现状**: 部分数据单位不够明确，如 TVS 显示为 "$42.1B"，但未说明是美元。

**建议**:

- 统一使用明确的货币符号
- 添加单位说明 tooltip
- 考虑添加货币切换功能
