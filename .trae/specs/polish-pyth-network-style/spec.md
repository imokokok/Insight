# Pyth Network 页面样式优化规范

## Why

当前 Pyth Network 页面功能完整，但在视觉层次、信息密度和样式一致性上还有轻微优化空间。基于现有 Dune 风格的扁平化设计语言，进行适度的样式优化，提升页面的专业感和可读性，同时避免过度设计。

## 现状分析

### 当前优点
1. **功能完整** - 8 个 Tab 覆盖市场、网络、发布者、验证者、跨链、价格源、风险、跨预言机对比
2. **Dune 风格基础** - 已使用 bg-dune 背景、扁平化设计
3. **数据丰富** - 发布者列表、验证者表格、跨链支持等多种数据展示
4. **交互友好** - 搜索、排序、悬停效果齐全

### 可优化点
1. **统计卡片样式不一致** - 页面使用了多种统计卡片样式（StatCard 和自定义样式混用）
2. **发布者卡片视觉层次** - 可以优化为更简洁的 Dune 风格
3. **验证者表格边框** - 可以简化为更轻量的分隔线设计
4. **颜色使用可以更克制** - 减少 violet-50 等强调色背景的使用
5. **区块间距需要统一** - 各区块间距略有差异

## What Changes

### 1. 统一统计卡片样式
- 统一使用 StatCard 组件，保持与 Chainlink 页面一致
- 移除自定义的统计卡片样式
- 保持 4 列网格布局

### 2. 优化发布者列表样式
- 将发布者统计卡片改为 FlatStatItem 风格
- 发布者卡片使用更简洁的边框设计
- 减少 violet 强调色的使用，改用中性灰色

### 3. 优化验证者表格样式
- 表格使用更轻量的分隔线
- 表头使用灰色背景替代白色
- 行悬停效果优化

### 4. 优化跨链支持卡片
- 统一卡片样式，使用 DashboardCard
- 减少装饰性元素

### 5. 优化价格源展示
- 使用统一的网格布局
- 简化图标和文字层级

## Impact

- 受影响文件：
  - `src/app/[locale]/pyth-network/page.tsx` - 主页面样式优化

## ADDED Requirements

### Requirement: 统一统计卡片
所有统计区域 SHALL 使用统一的 StatCard 组件

#### Scenario: 用户查看页面统计
- **WHEN** 用户查看页面顶部的统计数据
- **THEN** 看到统一风格的 4 列统计卡片
- **AND** 样式与 Chainlink 页面保持一致

### Requirement: 发布者区域 Dune 风格化
发布者列表 SHALL 使用更简洁的 Dune 风格设计

#### Scenario: 用户查看发布者列表
- **WHEN** 用户切换到 publishers tab
- **THEN** 看到简洁的统计概览
- **AND** 发布者卡片使用统一边框样式
- **AND** 减少强调色背景的使用

### Requirement: 验证者表格优化
验证者表格 SHALL 使用轻量分隔线设计

#### Scenario: 用户查看验证者列表
- **WHEN** 用户查看 validators tab 的表格
- **THEN** 表格使用简洁的分隔线
- **AND** 表头使用灰色背景
- **AND** 行悬停效果统一
