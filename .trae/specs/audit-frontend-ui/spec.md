# 前端页面UI/UX审查规范

## Why
作为一个专业的数据分析平台，Insight 的前端页面需要在视觉设计、用户体验、响应式布局和可访问性等方面达到专业水准。本次审查旨在发现并解决当前页面中存在的显示问题，提升整体用户体验。

## What Changes
- **布局与响应式问题修复**: 修复各页面的布局错位、溢出和响应式适配问题
- **视觉一致性优化**: 统一颜色、字体、间距等视觉元素
- **交互体验改进**: 优化按钮、表单、图表等交互组件
- **可访问性提升**: 改善键盘导航、屏幕阅读器支持等
- **性能优化**: 减少不必要的重渲染和布局抖动

## Impact
- 受影响页面: 首页、价格查询、跨链分析、跨预言机对比、市场概览、收藏、告警、设置、登录等
- 受影响组件: 导航栏、图表、表格、卡片、表单等核心组件

## ADDED Requirements
### Requirement: 布局与响应式修复
The system SHALL 修复以下布局问题:

#### Scenario: 首页布局问题
- **WHEN** 用户访问首页
- **THEN** 以下问题需要修复:
  1. ProfessionalHero 的搜索框在移动端显示过大，需要调整内边距和字体大小
  2. BentoMetricsGrid 的卡片在小屏幕上可能出现内容溢出
  3. LivePriceTicker 的跑马灯在窄屏幕上可能被截断

#### Scenario: 导航栏响应式问题
- **WHEN** 用户在移动端访问
- **THEN** 导航栏需要:
  1. Logo 和菜单按钮在极小屏幕上保持合适的间距
  2. 移动端抽屉菜单需要全屏显示以展示所有导航项
  3. 用户头像和下拉菜单在小屏幕上需要适配

#### Scenario: 图表页面布局
- **WHEN** 用户访问价格查询、跨链分析等含图表页面
- **THEN** 需要修复:
  1. 图表容器在移动端高度需要动态调整
  2. 图例文字过长时可能溢出容器
  3. 数据表格在小屏幕上需要水平滚动

### Requirement: 视觉一致性优化
The system SHALL 统一以下视觉元素:

#### Scenario: 颜色系统统一
- **WHEN** 页面渲染时
- **THEN** 需要统一:
  1. 所有页面的主色调应使用 `--finance-primary` (#1e40af)
  2. 成功/警告/错误状态色保持一致
  3. 渐变背景使用统一的渐变角度和颜色

#### Scenario: 字体和排版
- **WHEN** 文本渲染时
- **THEN** 需要:
  1. 统一标题层级（h1-h6）的字号和字重
  2. 正文文本使用一致的行高（1.5-1.75）
  3. 数字和货币使用等宽字体或tabular-nums

#### Scenario: 间距系统
- **WHEN** 组件布局时
- **THEN** 需要:
  1. 使用统一的间距刻度（4px、8px、12px、16px、24px、32px、48px）
  2. 卡片内边距保持一致（建议16px或24px）
  3. 页面最大宽度统一为 max-w-7xl (1280px)

### Requirement: 交互体验改进
The system SHALL 优化以下交互:

#### Scenario: 按钮和表单
- **WHEN** 用户与按钮/表单交互时
- **THEN** 需要:
  1. 按钮需要有明确的 hover、active、disabled 状态
  2. 表单输入框需要有清晰的 focus 状态指示
  3. 加载状态需要有统一的 loading 指示器

#### Scenario: 图表交互
- **WHEN** 用户与图表交互时
- **THEN** 需要:
  1. Tooltip 需要有足够的对比度和阴影
  2. 图表线条需要有足够的粗细（最小2px）
  3. 数据点 hover 效果需要明显

#### Scenario: 表格交互
- **WHEN** 用户与表格交互时
- **THEN** 需要:
  1. 表头需要固定（sticky）以便长表格滚动时可见
  2. 行 hover 效果需要明显但不突兀
  3. 排序指示器需要清晰可见

### Requirement: 可访问性提升
The system SHALL 提升以下可访问性:

#### Scenario: 键盘导航
- **WHEN** 用户使用键盘导航时
- **THEN** 需要:
  1. 所有交互元素需要有可见的 focus 轮廓
  2. Tab 顺序需要符合逻辑
  3. 快捷键需要文档化

#### Scenario: 屏幕阅读器支持
- **WHEN** 屏幕阅读器访问页面时
- **THEN** 需要:
  1. 图片需要有描述性的 alt 文本
  2. 图表需要有 ARIA 标签描述数据
  3. 动态内容更新需要通知屏幕阅读器

#### Scenario: 对比度和颜色
- **WHEN** 内容显示时
- **THEN** 需要:
  1. 文字与背景对比度至少达到 WCAG AA 标准（4.5:1）
  2. 不仅依赖颜色传递信息（同时使用图标、文字）
  3. 支持色盲友好的颜色模式

### Requirement: 性能优化
The system SHALL 优化以下性能:

#### Scenario: 图表渲染
- **WHEN** 图表数据更新时
- **THEN** 需要:
  1. 大数据集使用下采样（LTTB算法）
  2. 避免不必要的重渲染
  3. 动画使用 GPU 加速

#### Scenario: 图片和资源
- **WHEN** 页面加载时
- **THEN** 需要:
  1. 图片使用适当的尺寸和格式
  2. 使用懒加载延迟非关键资源
  3. 字体使用 font-display: swap
