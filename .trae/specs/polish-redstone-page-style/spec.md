# Redstone 预言机页面样式优化规格

## 评估总结

你的 Redstone 页面整体结构完整，功能丰富，已经很好地遵循了 Dune 风格的扁平化设计原则。以下是对当前实现的评估和适度的优化建议。

### 当前优点

1. **结构清晰**: 页面采用标准的标签页布局，包含市场数据、网络状态、数据提供者、数据流、跨链、生态系统和风险评估等多个维度
2. **组件复用**: 很好地使用了 DashboardCard、StatCard 等通用组件
3. **数据丰富**: 提供了全面的统计数据和指标展示
4. **风格一致**: 基本遵循了 Dune 扁平化设计（无圆角、边框分隔、简洁配色）

### 可优化点

1. **统计卡片区域**: 四个统计卡片的边框处理可以更加统一
2. **数据展示层次**: 部分数据展示缺乏视觉层次感
3. **空状态处理**: 某些数据加载状态的展示可以优化
4. **色彩运用**: Redstone 品牌色（红色）的运用可以更克制但更有目的性
5. **间距一致性**: 部分区块的间距可以进一步统一

## 优化方案

### 1. 统计卡片区域优化

**现状**: 使用 `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-4` 布局，每个 StatCard 内部有左边框分隔

**优化**:
- 采用 Dune 风格的连续边框设计，移除内部左边框，使用统一的网格边框
- 参考 `duneFlatDesign.ts` 中的 `stat-grid` 模式

### 2. 数据提供者列表优化

**现状**: 列表项使用 `bg-gray-50 rounded-lg` 背景

**优化**:
- 改为扁平化设计，使用边框分隔而非背景色区分
- 悬停效果改为边框颜色变化

### 3. 跨链支持列表优化

**现状**: 与数据提供者列表类似，使用圆角背景

**优化**:
- 统一为扁平化列表样式
- 状态标签（active/inactive）使用更克制的配色

### 4. 生态系统卡片优化

**现状**: 使用 `bg-gray-50 rounded-lg` 和分类标签

**优化**:
- 改为扁平化卡片设计
- 分类标签使用更 subtle 的样式

### 5. 风险评估面板微调

**现状**: RedStoneRiskAssessmentPanel 已经做得不错

**优化**:
- 进度条颜色统一使用 Redstone 主题色
- 评分趋势图表的参考线样式优化

## 影响范围

- **主要文件**:
  - `/src/app/[locale]/redstone/page.tsx`
  - `/src/components/oracle/panels/RedStoneRiskAssessmentPanel.tsx`

- **受影响组件**:
  - StatCard（可能需要添加新的变体）
  - DashboardCard（可能需要微调）

## 设计原则

1. **保持克制**: 不做过度设计，只在必要时添加视觉元素
2. **一致性优先**: 确保与项目中其他预言机页面风格统一
3. **功能优先**: 所有样式优化不得影响功能性和可读性
4. **渐进增强**: 从核心布局开始，逐步优化细节
