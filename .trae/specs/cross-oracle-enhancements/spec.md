# 跨预言机比较功能专业增强 - 产品需求文档

## Overview
- **Summary**: 对跨预言机比较功能进行深度专业强化，增加高级数据分析、统计指标、异常检测、数据导出和预言机健康监控等功能，打造专业级预言机数据分析平台。
- **Purpose**: 为 Web3 开发者、DeFi 分析师和预言机用户提供更专业的数据分析工具，帮助他们更准确地评估预言机可靠性、识别价格异常、进行深入的比较分析。
- **Target Users**: Web3 开发者、DeFi 分析师、预言机用户、量化交易者、风险管理专员

## Goals
- 增加高级统计指标（标准差、方差、一致性评级）
- 添加价格异常检测和高亮功能
- 实现基于置信度的加权平均计算
- 添加预言机健康状态监控（响应时间、可用性）
- 支持数据导出功能（CSV/JSON）
- 增加历史快照比较功能
- 添加自定义权重配置
- 优化图表交互，增加时间范围选择器
- 保持现有功能的完整性和稳定性

## Non-Goals (Out of Scope)
- 不添加新的预言机提供商
- 不修改数据获取的核心业务逻辑
- 不添加交易执行功能
- 不实现实时通知系统（邮件/短信）
- 不添加复杂的机器学习价格预测
- 不添加用户账户系统

## Background & Context
- 当前功能已具备基础比较能力，但缺乏专业级数据分析工具
- 用户需要更深入的统计分析来评估预言机可靠性
- 缺乏异常价格检测机制，可能导致用户错过重要信号
- 没有数据导出功能，不利于后续分析和报告生成
- 当前统计计算仅使用简单平均，未考虑置信度权重
- 缺少对预言机性能和健康状态的监控

## Functional Requirements
- **FR-1**: 添加高级统计指标（标准差、方差、一致性评级）
- **FR-2**: 实现价格异常检测和高亮显示（基于 Z-score）
- **FR-3**: 增加基于置信度的加权平均计算功能
- **FR-4**: 添加预言机健康状态监控（响应时间、可用性统计）
- **FR-5**: 实现数据导出功能（支持 CSV 和 JSON 格式）
- **FR-6**: 增加时间范围选择器（1小时/6小时/24小时/7天）
- **FR-7**: 添加自定义权重配置面板，允许用户调整各预言机权重
- **FR-8**: 优化图表，添加数据点标注和区域选择
- **FR-9**: 增加价格偏差警报阈值设置功能
- **FR-10**: 添加预言机可靠性评分系统

## Non-Functional Requirements
- **NFR-1**: 所有新增功能必须保持页面响应式设计
- **NFR-2**: 数据导出功能在 1000 条数据以内应在 2 秒内完成
- **NFR-3**: 异常检测算法执行时间应小于 100ms
- **NFR-4**: 新增样式应保持良好的可访问性（WCAG 2.0 AA 标准）
- **NFR-5**: 代码改动应保持可维护性和可测试性
- **NFR-6**: 页面加载时间增加不超过 15%
- **NFR-7**: 自动刷新间隔保持可选（不强制启用）

## Constraints
- **Technical**: 必须使用现有的技术栈（Next.js、Tailwind CSS、Recharts）
- **Business**: 功能增强应专业且实用，避免过度复杂化
- **Dependencies**: 依赖现有的预言机数据结构和 API 接口
- **Performance**: 不能显著影响页面响应速度
- **Browser Support**: 支持最新 2 个版本的 Chrome、Firefox、Safari 和 Edge

## Assumptions
- 用户具有基本的统计学知识，能够理解标准差、Z-score 等概念
- 用户希望通过更专业的工具来提升决策质量
- 数据导出功能对分析师用户有重要价值
- 用户关心预言机的可靠性和性能指标
- 基于置信度的加权平均能提供更准确的价格参考

## Acceptance Criteria

### AC-1: 高级统计指标显示
- **Given**: 用户已选择至少 3 个预言机
- **When**: 查看统计卡片区域
- **Then**: 显示标准差、方差和一致性评级（优秀/良好/一般/较差）
- **Verification**: `programmatic`
- **Notes**: 一致性评级基于标准差大小

### AC-2: 异常价格检测
- **Given**: 价格数据中存在离群值（Z-score > 2 或 < -2）
- **When**: 查看价格比较表格
- **Then**: 异常价格被高亮显示，并显示异常标记
- **Verification**: `programmatic`
- **Notes**: 使用 Z-score 方法检测异常

### AC-3: 加权平均计算
- **Given**: 用户已选择多个预言机且部分预言机有置信度数据
- **When**: 查看平均价格统计
- **Then**: 同时显示简单平均和加权平均两种计算结果
- **Verification**: `programmatic`

### AC-4: 预言机健康状态监控
- **Given**: 用户在跨预言机比较页面
- **When**: 数据加载完成后
- **Then**: 显示各预言机的响应时间和最近可用性统计
- **Verification**: `human-judgment`

### AC-5: 数据导出功能
- **Given**: 用户有可导出的价格数据
- **When**: 点击导出按钮并选择格式（CSV/JSON）
- **Then**: 数据文件成功下载，格式正确
- **Verification**: `programmatic`

### AC-6: 时间范围选择器
- **Given**: 用户查看价格趋势图表
- **When**: 选择不同的时间范围选项
- **Then**: 图表数据相应更新，显示所选时间范围内的数据
- **Verification**: `programmatic`

### AC-7: 自定义权重配置
- **Given**: 用户已选择多个预言机
- **When**: 用户打开权重配置面板并调整各预言机权重
- **Then**: 加权平均价格根据新权重重新计算
- **Verification**: `programmatic`

### AC-8: 价格偏差警报阈值
- **Given**: 用户设置了价格偏差警报阈值
- **When**: 某个预言机的价格偏差超过阈值
- **Then**: 该预言机在表格中被特别标记和高亮
- **Verification**: `programmatic`

### AC-9: 预言机可靠性评分
- **Given**: 有历史数据的预言机
- **When**: 查看预言机列表
- **Then**: 每个预言机显示可靠性评分（1-10分）
- **Verification**: `human-judgment`
