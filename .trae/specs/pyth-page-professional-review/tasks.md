## P0 - 高优先级任务（数据真实性）

### Task 1: 集成真实价格源数据
**描述**: 替换 PythPriceFeedsView 中的 mockPriceFeeds，使用 Pyth Hermes API 获取真实价格源列表
**优先级**: P0
**预估工时**: 8小时

**子任务**:
- [ ] 1.1 研究 Pyth Hermes API 的价格源列表接口
  - 阅读 Hermes API 文档
  - 确定获取所有价格源的端点
  - 了解价格源元数据结构

- [ ] 1.2 扩展 PythDataService 添加 getAllPriceFeeds 方法
  - 在 `src/lib/oracles/pythDataService.ts` 中实现
  - 支持分页和筛选参数
  - 添加错误处理和缓存机制

- [ ] 1.3 创建价格源数据 Hook
  - 在 `src/hooks/oracles/` 下创建 `usePythPriceFeeds.ts`
  - 实现数据获取、加载状态、错误处理
  - 支持自动刷新

- [ ] 1.4 更新 PythPriceFeedsView 组件
  - 替换 mockPriceFeeds 为真实数据
  - 添加加载状态展示
  - 处理空数据和错误状态

- [ ] 1.5 添加价格源实时更新
  - 集成 WebSocket 订阅价格变化
  - 实现价格闪烁动画效果
  - 显示最后更新时间

**验收标准**:
- 价格源列表显示真实数据，非 mock 数据
- 每个价格源显示实时价格、置信区间、更新频率
- 页面加载时有合适的加载状态
- 数据自动刷新，价格变化有视觉反馈

## 风险评估

| 风险 | 影响 | 可能性 | 缓解措施 |
|------|------|--------|----------|
| Pyth API 限流 | 高 | 中 | 实现请求队列和退避策略 |
| Solana RPC 不稳定 | 高 | 中 | 配置多个 RPC 端点，自动切换 |
| Dune Analytics 数据延迟 | 中 | 低 | 使用本地缓存作为备用 |
| 大数据量性能问题 | 中 | 高 | 提前进行性能测试和优化 |
| 跨链数据不一致 | 中 | 中 | 添加数据校验和异常处理 |
