# 跨预言机比较页面数据可视化优化 Spec

## Why
作为专业的数据分析平台，跨预言机比较页面需要提供更深入、更专业的数据可视化能力，帮助用户更好地理解不同预言机之间的价格差异、数据质量和性能表现。当前实现虽然功能完善，但在专业数据分析维度上还有优化空间，需要增强数据洞察力和分析深度。

## What Changes
- 增加价格偏差热力图，直观展示不同预言机在不同时间段的偏差分布
- 增加价格分布箱线图，展示价格分布特征和离群值
- 增加预言机价格相关性矩阵，分析不同预言机价格走势的相关性
- 增加数据质量评分卡片，综合展示数据新鲜度、完整性、可靠性
- 增加价格波动率对比图表，展示不同预言机的波动特征
- 增加延迟分布直方图，深入分析响应时间分布
- 优化现有图表的交互性和可读性
- 增加预言机性能排名可视化

## Impact
- Affected specs: 跨预言机比较页面数据分析能力、用户体验
- Affected code: 
  - `src/app/cross-oracle/page.tsx`
  - `src/components/oracle/CrossOracleComparison.tsx`
  - `src/i18n/zh-CN.json`
  - `src/i18n/en.json`

## ADDED Requirements

### Requirement: 价格偏差热力图
系统 SHALL 提供价格偏差热力图，直观展示不同预言机在不同时间段的偏差分布。

#### Scenario: 热力图展示
- **WHEN** 用户查看价格偏差分析时
- **THEN** 系统应显示热力图，横轴为时间，纵轴为预言机，颜色深浅表示偏差程度

#### Scenario: 热力图交互
- **WHEN** 用户悬停在热力图单元格上时
- **THEN** 系统应显示具体的时间、预言机名称、偏差百分比和价格

### Requirement: 价格分布箱线图
系统 SHALL 提供价格分布箱线图，展示各预言机价格的统计分布特征。

#### Scenario: 箱线图展示
- **WHEN** 用户查看价格分布时
- **THEN** 系统应显示箱线图，包含中位数、四分位数、离群值等统计信息

#### Scenario: 离群值标注
- **WHEN** 箱线图中存在离群值时
- **THEN** 系统应以特殊标记显示离群值点，并标注具体价格

### Requirement: 预言机价格相关性矩阵
系统 SHALL 提供预言机价格相关性矩阵，分析不同预言机价格走势的相关程度。

#### Scenario: 相关性计算
- **WHEN** 用户查看相关性分析时
- **THEN** 系统应计算并显示各预言机之间的皮尔逊相关系数

#### Scenario: 相关性可视化
- **WHEN** 显示相关性矩阵时
- **THEN** 系统应使用颜色渐变表示相关程度，并提供数值标注

### Requirement: 数据质量评分卡片
系统 SHALL 提供综合数据质量评分卡片，展示数据新鲜度、完整性、可靠性等指标。

#### Scenario: 数据新鲜度
- **WHEN** 用户查看数据质量时
- **THEN** 系统应显示最后更新时间距现在的时长，并用颜色指示新鲜程度

#### Scenario: 数据完整性
- **WHEN** 用户查看数据质量时
- **THEN** 系统应显示数据完整率（成功获取数据的预言机占比）

#### Scenario: 数据可靠性
- **WHEN** 用户查看数据质量时
- **THEN** 系统应显示数据可靠性评分（基于历史准确率和响应成功率）

### Requirement: 价格波动率对比
系统 SHALL 提供价格波动率对比图表，展示不同预言机的价格波动特征。

#### Scenario: 波动率计算
- **WHEN** 用户查看波动率分析时
- **THEN** 系统应计算并显示各预言机价格的标准差、变异系数

#### Scenario: 波动率趋势
- **WHEN** 用户选择时间范围时
- **THEN** 系统应显示该时间段内波动率的变化趋势

### Requirement: 延迟分布直方图
系统 SHALL 提供延迟分布直方图，深入分析各预言机的响应时间分布。

#### Scenario: 延迟分布展示
- **WHEN** 用户查看延迟分析时
- **THEN** 系统应显示响应时间的频率分布直方图

#### Scenario: 延迟统计指标
- **WHEN** 显示延迟分布时
- **THEN** 系统应标注 P50、P95、P99 延迟值

### Requirement: 预言机性能排名
系统 SHALL 提供预言机性能排名可视化，综合评估各预言机的表现。

#### Scenario: 综合评分
- **WHEN** 用户查看性能排名时
- **THEN** 系统应根据响应时间、准确率、稳定性等指标计算综合评分并排名

#### Scenario: 排名趋势
- **WHEN** 用户查看历史排名时
- **THEN** 系统应显示排名变化趋势

### Requirement: 图表交互优化
系统 SHALL 优化现有图表的交互性和可读性。

#### Scenario: 图表缩放
- **WHEN** 用户查看价格趋势图时
- **THEN** 系统应支持图表缩放和平移功能

#### Scenario: 数据点高亮
- **WHEN** 用户悬停在图表数据点上时
- **THEN** 系统应高亮显示该数据点并展示详细信息

#### Scenario: 图表导出
- **WHEN** 用户需要保存图表时
- **THEN** 系统应支持将图表导出为 PNG 或 SVG 格式

## MODIFIED Requirements

### Requirement: 现有统计指标增强
在现有统计指标基础上，增加趋势指示和对比分析。

**增强内容**：
- 增加指标变化趋势箭头（与上次刷新对比）
- 增加指标历史最小/最大值标注
- 增加指标健康度颜色编码

### Requirement: 现有价格表格增强
在现有价格比较表格基础上，增加更多专业分析维度。

**增强内容**：
- 增加价格偏离度颜色编码（绿色<0.1%，黄色0.1-0.5%，红色>0.5%）
- 增加数据新鲜度指示器（显示距上次更新的时间）
- 增加快速筛选功能（按偏差范围筛选）

### Requirement: 现有图表增强
在现有价格趋势图基础上，增加更多分析工具。

**增强内容**：
- 增加价格偏差通道（显示平均价格±标准差范围）
- 增加数据更新事件标记点
- 增加图表工具栏（缩放、重置、导出）

## REMOVED Requirements
无移除的需求。所有现有功能保留并增强。

## 专业优化原则

### 设计原则
1. **数据优先**：所有可视化应以数据准确性为首要目标
2. **简洁明了**：避免过度装饰，突出关键信息
3. **交互友好**：提供直观的交互方式，降低学习成本
4. **响应式设计**：确保在不同设备上都有良好的展示效果

### 实施优先级

**高优先级（核心功能）**：
1. 价格偏差热力图 - 直观展示偏差分布
2. 数据质量评分卡片 - 综合评估数据质量
3. 价格分布箱线图 - 展示价格分布特征
4. 延迟分布直方图 - 深入分析响应时间

**中优先级（增强功能）**：
5. 预言机价格相关性矩阵 - 分析价格相关性
6. 价格波动率对比 - 展示波动特征
7. 预言机性能排名 - 综合性能评估
8. 图表交互优化 - 提升用户体验

**低优先级（可选功能）**：
9. 图表导出功能
10. 历史排名趋势

### 技术实现建议
1. 使用 Recharts 库现有的图表组件
2. 自定义 Tooltip 和 Legend 以提供更丰富的信息
3. 使用 CSS Grid 和 Flexbox 实现响应式布局
4. 使用 Web Workers 处理大量数据计算（如相关性矩阵）
5. 使用 Canvas 渲染大数据量的热力图（如需要）
